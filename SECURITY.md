# Security Policy for API-Waygate 🔒

Security is a top priority for API-Waygate and its users. Please read this policy to understand how we handle security issues.

---

## Supported Versions

We actively maintain the latest version of API-Waygate.  

To help keep your applications secure:

- Regularly update to the newest release.
- Write and run tests for your integrations before upgrading.
- Check release notes for bug fixes and security improvements.

Keeping your dependencies up to date ensures you benefit from the latest features and security patches.

---

## Reporting a Vulnerability

If you discover a potential security issue in API-Waygate, please report it immediately. Even if you’re unsure, we appreciate your help.

- Email: **richardquaicoe78@gmail.com**  
- Include as much detail as possible:
  - Steps to reproduce the issue
  - Example code if applicable
  - Environment details (version, OS, etc.)

All reports will be reviewed promptly and treated confidentially. The maintainer will get back to you with guidance and next steps.

---

## Responsible Disclosure

- Do **not** discuss potential security vulnerabilities publicly until a fix has been released.  
- Private reporting helps minimize the risk of exploitation.  

We aim to address issues quickly and keep both users and the community safe.  

---

Thank you for helping make API-Waygate secure! 🙏
